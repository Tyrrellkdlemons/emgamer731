## 4. ROUTING DESIGN — how the new endpoints fit together

```
┌─────────────────────────────────────────────────────────────────────┐
│   Browser                                                           │
│                                                                      │
│   /watch ──server-render──► getYouTubeFeed()                        │
│   home rail ─SWR poll────► /api/youtube ────► getYouTubeFeed()      │
│   /eats ────────────────► EATS_TIKTOKS (static, curated)            │
│   live banner ─SWR poll► /api/live ───► getLiveSummary() ──┬────── youtubeAdapter (Data API)
│                                                              └────── tiktokAdapter (reads tiktok-store)
│                                                                                │
│                                                                                ▼
│                                          tiktok-store (in-memory, 30-min TTL)
│                                                                                ▲
│   TikTok Live for Business ──webhook──► /api/webhooks/tiktok-live ─────────────┤
│   iPhone Shortcut ────POST/DEL bearer► /api/admin/live-override ───────────────┤
│   MANUAL_LIVE_OVERRIDE env var ──────► tiktokAdapter direct read ──────────────┘
└─────────────────────────────────────────────────────────────────────┘
```

Key invariants:
- **Visitors never leave the site to watch**: every video/short embeds inline via the facade-pattern player.
- **Auto-pull cadence**: ~30 min for YouTube, 60 s for live status. Server-side cache + ISR keep YouTube quota usage at ~5K units/day max.
- **Live trigger has 3 redundant paths**: webhook → admin override → env-var fallback. If one fails, the next picks up.
- **TikTok role is narrow by design**: only live-Roblox + curated food posts. Bulk TikTok import never happens.

## 5. COST / LATENCY / RISK

**Cost:** $0/month if shop is brochure-only. With shop fulfillment via Snipcart, ~2% per transaction. Optional `YOUTUBE_API_KEY` quota is free up to 10K units/day (we use ~5K). Optional Replicate-style premium image gen would add per-render cost (deferred).

**Latency targets (Lighthouse mobile, simulated 4G, Moto G4):**
- LCP: < 1.8s (hero photo + Bricolage Grotesque + Plus Jakarta Sans preloaded)
- Performance: ≥ 92
- Accessibility: ≥ 96
- Best Practices: 100
- SEO: 100

**Risk register:**
- *YouTube API quota exhaustion.* Mitigation: 30-min cache + ISR; only ~5K units/day used.
- *YouTube channel deletes RSS feed* (already does — handled gracefully via fallback chain).
- *TikTok webhook never fires.* Mitigation: admin override + env-var override.
- *Real avatar image deleted from `/public`.* Mitigation: SVG fallback in `AvatarCard.tsx` renders automatically.
- *Domain DNS misconfigured.* Mitigation: Netlify `*.netlify.app` URL stays live.
- *Shop fulfillment errors.* Not a risk on day 1 — shop is presentation-only.
